Website Redesign

Heads Up! I wrote this just after laumch when I still worked at the SU. Who knows what has happened since!

Overview

Over the last two years I have been working for the Students’ Union I have had a healthy dose of variety in my job. Around a central set of events it is amazing how much variance gets into the day-to-day activity.

One common element over the past year and a half was the website theme. As a developer by trade this was going to be my ‘masterpiece’ or so I hoped. If any project was going to be it it was this.

Why did we need to do this?

The old site had not aged well. Unresponsive, grey and difficult to work with for day-to-day web work. The plan was to install a new responsive, well documented framework. With the aim speed up future development work.

I settled on Foundation 5 from Zurb. At the time it was the most up-to-date version of their responsive framework. Halfway into work they launched Foundation 6 (a plan to update to six is currently in the works).

One of the main challenges was managing code between production, staging, and development. With source code entered in the browser, I needed to find a way to use my workflow to produce high quality code. Then move it to staging to test. Using Coda and CodeKit to manage local code changes, then copied the minified code to the browser. Once happy with a feature or development, commit to the repo and move to the next todo.

What makes up an MSL Skin?

Dust, bits of grit and a whole lot of duct tape:

Snippets – Bits of reusable HTML that have fragmented functionality. I.e – the html head content that is used on each page. Using a snippet means you can reuse the same code on many pages.

Templates – These are the page templates themselves. Using code to layout what pages look like. Setting where the page content lives (these are the only things that don’t live in the repo at the moment).

Site Assets / Globals – These are scripts and styles that are uploaded to the site. Foundation core modules live in the site assets area.

The redevelopment received with much rejoicing. No quite as python but the small amount of feedback I have had has been positive. Online engagement have increased but the website is just the delivery channel. The content is what is on the cards for improving next. I’m now focusing on building links with our IT and design courses to get them involved with our design work.
